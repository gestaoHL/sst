Como deve ser organizada a navegação da plataforma?

Para um SST de metrô com múltiplos módulos, a estrutura de navegação define a experiência do usuário diário.

A

Sidebar Lateral Fixa + Dashboard Inicial

Painel de KPIs na abertura (acidentes, ASOs vencendo, EPIs vencidos). Sidebar com todos os módulos sempre visível. Padrão das grandes plataformas corporativas (SAP, TOTVS SST, eSocial).

Abas Superiores por Área (atual, expandido)

Mantém a navegação horizontal por abas, expandindo para ~8 módulos. Mais familiar para o time atual, menor curva de aprendizado. Mais simples visualmente mas fica congestionado com muitos módulos.

Menu Agrupado por Domínio (dois níveis)

Sidebar com grupos colapsáveis: Saúde agrupa ASO/PCMSO/Absenteísmo; Segurança agrupa Acidentes/Inspeções/APR. Escala melhor para muitos módulos, hierarquia clara. Padrão de sistemas como GHE, SESI SST.
